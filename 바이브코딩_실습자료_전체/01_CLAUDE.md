# 프로젝트 규칙 (AI가 매번 먼저 읽는 쪽지)
# 안드레이 카파시 4원칙 + 한국어 실전 5규칙. 그대로 복사해서 프로젝트 폴더에 CLAUDE.md로 저장하세요.

## 원칙 1. 코딩하기 전에 먼저 생각한다
- 가정하지 말고, 애매하면 **먼저 질문**한다.
- 해석이 여러 개면 다 보여주고, 더 단순한 방법이 있으면 말한다.
- 모르는 게 있으면 멈추고 무엇이 헷갈리는지 묻는다.

## 원칙 2. 단순하게 먼저
- 문제를 푸는 **최소한의 코드**만 쓴다.
- 시키지 않은 기능·추상화·설정·불필요한 에러 처리를 넣지 않는다.
- 200줄이 50줄로 될 것 같으면 다시 쓴다.

## 원칙 3. 외과수술처럼 정확하게
- **시킨 것만** 고친다. 옆에 있는 코드·주석·서식을 함부로 바꾸지 않는다.
- 안 망가진 것을 리팩토링하지 않는다. 기존 스타일에 맞춘다.
- 바뀐 모든 줄은 사용자의 요청으로 거슬러 올라갈 수 있어야 한다.

## 원칙 4. 목표 중심으로 실행
- 성공 기준을 **테스트**로 정하고, 통과할 때까지 반복한다.
- ‘검증 추가해줘’ 대신 → ‘예상치 못한 입력 테스트를 만들고 통과시켜줘’처럼 준다.
- 명령하지 말고, 성공 기준을 주고 지켜본다.

## 한국어 실전 5규칙
1. 한국어 문장은 콜론(:)이 아니라 **마침표(.)** 로 끝낸다. (코드·라벨은 예외)
2. 새 소스 파일 첫 줄에 **그 파일이 무엇을 하는지 한 줄 한글 주석**을 넣는다.
3. 큰 작업은 **계획서 → 체크리스트 → 컨텍스트 노트** 3종 문서를 먼저 만든다.
4. 코드를 고친 세션에서는 ‘다 됐어’ 말하기 전에 **반드시 테스트를 한 번 돌린다.**
5. 의미 있는 작업이 끝나면 **바로 커밋**한다. (커밋 메시지를 한 문장으로 설명 가능하면 커밋)
