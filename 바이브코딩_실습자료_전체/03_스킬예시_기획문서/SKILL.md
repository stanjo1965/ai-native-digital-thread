# UX 기획 문서 자동 생성 스킬
사용자가 앱/서비스 주제를 주면, 아래 3종 문서를 순서대로, 한국어로, 초보자도 이해하게 작성한다. 어려운 말은 괄호로 쉬운 설명을 붙인다.

## 1) 요구사항 명세서 (무엇을 만드나)
- 앱 이름/한 줄 소개, 목적(어떤 문제를 푸나), 핵심 사용자, 사용자 가치 3가지, 핵심 기능 5~8개, 범위 밖(안 만들 것).

## 2) 기능 명세서 (어떻게 동작하나)
- 기능마다: 동작 순서(사용자가 …누르면 …된다), 입력 항목, 예외/규칙. 추가·조회·수정·삭제(CRUD)를 각각 다룬다. 삭제 전 "정말 삭제할까요?" 확인 포함.

## 3) IA 정보구조도 (화면 지도)
- 화면 목록, 화면 이동(트리로), 메뉴 구조(하단 탭 등).

## 마무리
세 문서를 다 쓴 뒤 "이 문서를 Google Stitch에 넣으면 화면 디자인을 만들 수 있어요"라고 안내한다.
