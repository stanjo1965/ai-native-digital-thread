# 🚀 AI 개발 4단계 따라하기 — CLAUDE.md → 스킬 → 하네스 → 루프
> 초등학생도 그대로 따라할 수 있게, 한 줄씩 천천히! 다 하면 ✅
> 입력하는 곳은 3군데예요. 헷갈리지 않게 **[터미널] · [편집기] · [클로드 코드]** 표시를 봐요.

## 어디에 입력하나요? (3군데)
| 표시 | 무엇 | 언제 |
|------|------|------|
| **[터미널]** | 검은 명령창(윈도우: 시작→‘PowerShell’) | 폴더 만들기·앱 실행 |
| **[편집기]** | 메모장 또는 VS Code | 파일 만들고 저장 |
| **[클로드 코드]** | 클로드 코드 채팅창 | AI에게 부탁하기 |

## 준비물
- 인터넷 되는 컴퓨터 · 구글/이메일 계정(부모님 도움) · 클로드 코드 설치(터미널·앱 아무거나) · 약 40분

---

# 🟦 STEP 1. CLAUDE.md — ‘규칙 쪽지’ 만들기  (10분)
> AI는 똑똑하지만 매일 기억을 잃는 신입이에요. 그래서 **규칙 쪽지**를 붙여줘요.
> AI는 일 시작 전에 이 쪽지를 항상 먼저 읽어요.

- [ ] **1-1.** [터미널] 내 프로젝트 폴더를 만들고 들어간다.
  ```
  mkdir C:\내앱
  cd C:\내앱
  ```
- [ ] **1-2.** [편집기] 이 폴더에 새 파일 **`CLAUDE.md`** 를 만든다.
- [ ] **1-3.** 준비된 **`01_CLAUDE.md`** 파일 내용을 복사해서 붙여넣고 저장(Ctrl+S)한다.
  > (전 세계가 쓰는 카파시 4원칙 + 한국어 5규칙이 들어 있어요.)
- [ ] **1-4.** 끝! 이제 AI는 이 폴더에서 항상 이 규칙을 지켜요.

> 💡 규칙 쪽지가 하는 일: “가정하지 말고 물어봐 · 짧게 만들어 · 시킨 것만 고쳐 · 다 되면 테스트해”

---

# 🟩 STEP 2. 스킬(Skill) — ‘자동 버튼’ 만들기  (10분)
> 자주 시키는 일을 **버튼 하나**처럼 만들어 둬요. 긴 부탁을 매번 안 써도 돼요.

- [ ] **2-1.** 준비된 **`03_스킬예시_SKILL.md`** 파일을 확인한다. (기획 문서를 자동으로 만들어주는 스킬)
- [ ] **2-2-A.** [클로드 코드(터미널·앱)] 스킬 폴더에 넣기:
  ```
  # 내 컴퓨터 전체에서 쓰기
  ~/.claude/skills/기획문서/SKILL.md  위치에 파일을 둔다
  ```
- [ ] **2-2-B.** (claude.ai 웹/앱을 쓰면) **설정 → 사용자 지정 → 스킬 → ‘+’ → 스킬 업로드**로 올린다.
- [ ] **2-3.** [클로드 코드] 이렇게만 말하면 스킬이 저절로 켜진다:
  ```
  버킷리스트 앱 기획 문서 만들어줘
  ```
- [ ] **2-4.** 요구사항·기능·화면지도 문서가 뿅! 나온다.

> 💡 스킬 파일 규칙: 맨 위에 이름·설명(YAML)이 있어야 하고, 폴더로 넣을 땐 `SKILL.md` 이름이어야 해요.

---

# 🟧 STEP 3. 하네스(Harness) — ‘AI 팀’ 만들기  (10분)
> 혼자 일하는 AI 대신, **여러 전문가 AI(설계·개발·검사)가 한 팀**이 되게 해요.

- [ ] **3-1.** [편집기] ‘에이전트 팀’ 기능을 켠다.
  파일: `~/.claude/settings.json` (준비된 **`02_settings.json`** 내용을 넣기)
  ```
  { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
  ```
  → 저장 후 **클로드 코드를 껐다 켠다**.
- [ ] **3-2.** [클로드 코드] 하네스 플러그인을 설치한다.
  ```
  /plugin marketplace add revfactory/harness
  /plugin           (Discover 탭 → harness 선택 → 설치)
  /reload-plugins
  ```
- [ ] **3-3.** [클로드 코드] 한국어로 딱 한 줄:
  ```
  하네스 구성해줘
  ```
- [ ] **3-4.** 내 일에 맞는 **AI 팀(아키텍트·개발·QA)**이 자동으로 만들어진다.

> 💡 하네스가 하는 일: 규칙(CLAUDE.md)·검사(훅)·매뉴얼(스킬)·팀(에이전트)을 한데 묶어 ‘잘 일하는 환경’을 만든다.

---

# 🟪 STEP 4. 루프(Loop) — ‘완성될 때까지 자동 반복’  (10분)
> AI가 스스로 만들고·검사하고·고치기를 **완성될 때까지** 반복해요.
> ⚠️ 제일 중요한 건 **‘언제 멈출지(완료 기준)’**를 정하는 것! (안 그러면 영원히 돌아요.)

- [ ] **4-1.** [클로드 코드] 루프 엔진(공식 랄프 루프) 설치.
  ```
  /plugin           (install → ralph-loop → repository-only)
  /reload-plugins
  ```
- [ ] **4-2.** [클로드 코드] 먼저 계획부터 (Shift+Tab → Plan 모드):
  ```
  현재 하네스는 구축된 상태다. 루프를 어떻게 할지 계획만 세워봐.
  ```
  AI가 물으면 정한다 → **완료 기준**: 테스트·QA·기능체크 다 통과 / **자율성**: 감독된 반복(5~10회) / **최대 30회**
- [ ] **4-3.** 계획을 읽고 좋으면 실행:
  ```
  캘린더 기반 할일앱을 루프 엔지니어링으로 개발해줘
  ```
- [ ] **4-4.** AI 팀이 스스로 반복하며 완성한다. 끝나면 `progress.md`에 전 과정이 기록된다.

> 💡 준비된 **`04_루프_프롬프트_템플릿.md`** 와 **`05_PRD_예시_캘린더할일앱.md`** 를 함께 쓰면 더 쉬워요.
