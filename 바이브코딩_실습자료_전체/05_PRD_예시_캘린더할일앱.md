# PRD (완료 기준) 예시 — 캘린더 기반 할일 앱
> 이 파일을 `loop/acceptance.md`로 두면, 루프가 이 목록을 하나씩 완료할 때까지 반복합니다.
> 체크박스가 전부 [x]가 되고 3종 게이트를 통과하면 루프가 종료됩니다.

## 완료 기준 (Acceptance Criteria)
- [ ] **캘린더 뷰 전환**: 월간 / 주간 / 일간 보기를 전환할 수 있다.
- [ ] **할일 생성(Create)**: 날짜를 골라 할일(제목·시간·메모)을 추가할 수 있다.
- [ ] **할일 조회(Read)**: 날짜별로 등록된 할일 목록을 볼 수 있다.
- [ ] **할일 수정(Update)**: 기존 할일의 내용을 고칠 수 있다.
- [ ] **할일 삭제(Delete)**: 확인 창을 거쳐 할일을 삭제할 수 있다.
- [ ] **완료 토글**: 할일을 완료 처리(체크)하고 되돌릴 수 있다.
- [ ] **반복 일정**: 매일/매주 반복되는 할일을 설정할 수 있다.
- [ ] **타임존 처리**: 사용자 지역 시간에 맞게 날짜·시간이 정확히 표시된다.
- [ ] **알람/리마인더**: 지정 시각에 알림이 뜬다.

## 검증(3종 게이트) — 모두 통과해야 종료
1. **자동 검증**: 테스트 · 빌드 · 타입체크 · 린트 전부 통과
2. **QA 리포트**: 위 각 기능 PASS + 판정 근거 기록
3. **기능 체크리스트**: 위 완료 기준 항목이 모두 [x]

## 안전장치
- **Max Iteration = 30** (무한 루프 방지, 초과 시 강제 종료·보고)
- 이터레이션 단위 = **완료 기준 항목 1개**

## 범위 밖(이번엔 안 만듦)
- 로그인/계정, 팀 공유, 클라우드 동기화 — 다음 버전에서.
