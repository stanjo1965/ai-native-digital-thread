# 루프 프롬프트 원문(템플릿) — 매 사이클 ‘그대로’ 주입되는 프롬프트
> 루프의 심장. 랄프 루프가 기동되면 **매 반복마다 이 프롬프트 전체가 그대로 주입**됩니다.
> ⚠️ 이 프롬프트는 임의로 요약·변경하지 않습니다. (workspace·loop 상태만 바뀌고, 프롬프트 원문은 고정)
> `<대괄호>` 부분만 여러분의 프로젝트에 맞게 바꿔 쓰세요.

---

당신은 `<프로젝트: 예) 캘린더 기반 할일(To-do) 앱>`을(를) **완료 기준을 충족할 때까지 반복 개발하는 루프**를 수행한다.

## 매 사이클마다 하는 일
1. **컨텍스트 확인**: `loop/progress.md`(진행 기록)와 `loop/acceptance.md`(PRD·완료 기준)를 먼저 읽는다. 지금까지 무엇을 했고, 남은 것이 무엇인지 파악한다.
2. **다음 작업 선택**: PRD의 완료되지 않은 항목 중 **가장 우선인 기능 1개**를 이번 이터레이션의 목표로 잡는다.
3. **구현**: 하네스의 오케스트레이터/서브에이전트(아키텍트·백엔드·프론트엔드·QA)를 호출해 구현한다. 에이전트끼리 교차 검증한다.
4. **검증(3종 게이트)**: 아래를 모두 실행한다.
   - 자동 검증: 테스트 · 빌드 · 타입체크 · 린트
   - QA 리포트: 기능별 판정(PASS/FAIL)과 근거를 남긴다.
5. **기록**: 이번 이터레이션에서 무엇을 했는지, 실패·수정 내역을 `loop/progress.md`에 남긴다. (가능하면 git commit)

## 종료(완료) 판정
- **모두 그린**(3종 게이트 통과 + PRD 전 항목 완료)이면 `COMPLETE`를 출력하고 **루프를 종료**한다.
- 실패했거나 PRD에 미완료 스토리가 남아 있으면 **다음 반복**으로 넘어간다.
- 단, **최대 반복 횟수(Max Iteration = `<예: 30>`)**를 넘기면 무한 루프 방지를 위해 **강제 종료**하고 남은 항목을 보고한다.

## 규칙
- 이 프롬프트 내용을 임의로 요약하거나 바꾸지 않는다.
- ‘느낌상 끝난 것 같다’로 종료하지 않는다. **반드시 검증 근거(증거)로 판정**한다.
- 각 반복은 작게(기능 1개 단위) 유지한다.

- `loop/progress.md` — 이터레이션별 진행 기록(자동 작성됨).
