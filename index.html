PART 0 · 개념

Digital Thread란 무엇인가

제조·PLM에서 온 개념 — 전 과정을 하나의 실로 잇는다

원래 의미(제조·PLM)

  • • 설계→생산→운영까지 데이터를 하나로 연결
  • • 모든 산출물이 최초 요구로 ‘추적’된다
  • 끊긴 정보가 아니라 연속된 흐름

AI 개발에 적용하면

  • • 요구·결정 → CLAUDE.md 규칙 → 코드
  • • → 검증(훅·QA) → 문서(약관·방침) → 운영
  • 모든 변경이 최초 ‘결정’으로 추적된다
정리하면 카파시 원칙 ③ ‘모든 변경된 줄은 요청으로 거슬러 올라갈 수 있어야 한다’ = Digital Thread의 정신.

🔎 웹 보강 · Digital Thread란(제조·PLM 원류)

  • 제조/PLM에서 ‘디지털 스레드’는 설계→생산→운영의 모든 데이터를 하나의 실로 이어 추적성(traceability)을 보장하는 개념이다.
  • AI 개발에 옮기면: 요구·결정 → CLAUDE.md 규칙 → 코드 → 검증(훅·QA) → 문서 → 운영까지 끊기지 않게 잇고, 모든 산출물이 최초 결정으로 거슬러 올라간다.
  • 카파시 원칙 ③ ‘모든 변경된 줄은 요청으로 추적 가능해야 한다’ = 디지털 스레드의 정신.

PART 1 · 왜 하네스인가

AI는 코드를 못 짜는 게 아니라 너무 잘 짜는 게 문제

카파시가 짚은 3대 고질병과 그 브레이크

AI의 3대 고질병

  • ① 잘못된 가정 — 묻지 않고 JWT 200줄
  • ② 코드 부풀리기 — 3줄 일을 200줄로
  • ③ 멋대로 고치기 — 시킨 것 외까지 손대 망가짐
  • • ‘너무 빨리·자신있게·묻지 않고’ 짠다

브레이크 = CLAUDE.md 65줄

  • ① 코딩 전에 생각(가정 말고 물어라)
  • ② 단순하게 먼저(200줄→50줄)
  • ③ 외과수술식(시킨 것만)
  • ④ 목표 주도(성공 기준=테스트)
정리하면 forrestchang의 65줄 CLAUDE.md는 깃헙 스타 10만+. + datajuny 한국어 커스텀 5(마침표·역할주석·3종 문서·테스트·의미 커밋).

🔎 웹 보강 · 카파시 원칙의 최신 근거

  • 원칙의 출처: 카파시가 2025년 말 X에 올린 ‘LLM 코딩의 함정’ 관찰을 개발자 Forrest Chang이 4원칙 CLAUDE.md(약 60줄, 깃헙 스타 10만+)로 정리했다.
  • 실측 효과: 이 규칙을 적용하자 Claude의 코딩 오류율이 41% → 11%로 떨어졌다는 보고가 있다.
  • 2026년 6월, 원본 CLAUDE.md는 10개 규칙으로 확장되며 ‘자기점검(self-check) 프로토콜’이 추가됐다 — 루프 중 스스로 요구를 재확인하게 만든다.

PART 2 · 가장 쉬운 설명

AI는 ‘매일 기억을 잃는 똑똑한 신입’입니다

하네스는 그 신입이 실수 없이 일하게 해주는 ‘업무 환경 세트’예요. 비유가 어렵다면 이 한 문장만 기억하세요.

하네스가 없으면 (매일 새 신입)

  • 우리 회사 규칙을 모른다
  • 어제 한 일을 기억 못 한다
  • ‘하지 마’라고 해도 또 한다
  • 결국 내가 매번 확인하고 고친다

하네스가 있으면

  • 붙여둔 규칙을 먼저 읽고 시작한다
  • 실수하면 자동으로 걸려서 못 넘어간다
  • 정해진 순서·담당 동료와 협업한다
  • 내가 안 챙겨도 우리 방식대로 한다
정리하면 똑똑함(AI 모델)은 그대로 두고, ‘일하는 환경’만 깔아주는 것 — 그게 하네스입니다.

PART 3 · 정의

‘나는 프롬프트를 쓰지 않는다, 루프를 작성한다’

인간은 목표만 — 나머지는 AI가 반복한다

기존 방식 (인간이 병목)

  • • 프롬프트 입력 → AI 실행·결과
  • 인간이 테스트 → ‘고쳐줘’ 재요청
  • AI 수정 → 인간 확인 → 반복
  • 인간의 확인 속도가 병목

루프 방식 (인간은 목표만)

  • 인간: 목표(성공 기준)만 설정
  • AI 스스로 프롬프트·검증·수정
  • 목표 성공까지 무한 반복
  • 하네스(환경)가 있어야 안전하게 돈다
정리하면 ⚠️ 토큰 소모가 크다 — 목표·종료조건을 명확히 하고 필요할 때만 쓴다.

🔎 웹 보강 · 랄프 루프(Ralph Loop)의 정체

  • 창안: Geoffrey Huntley. 이름은 심슨 캐릭터 ‘랄프 위검’에서 왔다 — 단순하지만 끈질기게 같은 일을 반복한다는 뜻.
  • 구조: while true 로 같은 프롬프트 파일을 매번 새 컨텍스트로 반복 투입 → 코드 읽기·작업·커밋·종료를 계속.
  • 3단계: ① 발견(대화로 탐색) → ② 명세(PRD·작업목록) ← 성공을 가르는 최대 변수 → ③ 실행(명세 참조 반복) + 재검증(요구 충족 여부 스스로 확인).
  • 비용·규모: Claude를 배시 루프로 돌리면 대략 시간당 $10, 하룻밤에 1,000+ 커밋도 나온다 — 그래서 종료조건이 필수다.
